Travel Planner Z · 產品設計說明書

把一場複雜旅行的所有細節，
裝進一個一目了然的工具。

Travel Planner Z 是為「真的會在地圖上拖拉、會比較三個版本、會印成紙本帶上路」的旅人設計的行程規劃工作台。它整合了景點搜尋、時程編排、地圖距離、多方案費用對比、AI 行前建議與旅遊手冊匯出於一頁之中，避免使用者在 Google My Maps、Notion 和 Excel 之間反覆貼上資料。

產品階段已上線（Phase 15+）
部署Vercel · Neon Postgres
使用模式個人優先 · 可分享協作
介面語言中文（混合英文術語）

01 · 產品概念

它解決什麼問題？

市面上每個工具都只做對一件事：Google Maps 會帶路但記不住計畫，Notion 寫得了筆記但畫不出地圖，Excel 算得了帳但拖不動景點。一場跨日、跨城、跨方案的旅行，使用者最後總是要在三個視窗之間複製貼上，並且自己心算油錢與匯率。

🗺

排程 + 地圖一氣呵成

左邊拖時間，右邊看路線，景點在何處、誰先誰後、走過去要幾分鐘，全部在同一個畫面內反應。

🔀

多方案並列對比

同一趟旅行可以開三個版本（輕鬆版／緊湊版／省錢版），並肩看一眼就知道哪天哪個方案最划算。

💴

真實匯率與油錢試算

每筆花費自動換成主幣，駕車段自動估油費，所有 ticket 都同步進費用總覽，不需要重輸入。

📖

一鍵印出帶在身上

選紙張、選方向、選章節，即時預覽多頁手冊，直接生成 PDF。網路斷了照樣有東西看。

02 · 目標使用者

誰會用它？

這不是一個給「兩天兩夜小週末」的人用的工具。它的價值在計畫越複雜、變數越多時越突出。

🧳 重度自助旅行者

一趟 7–14 天、跨多個城市、會自駕也會搭電車、會排到 6 個方案的人。對「省 30 分鐘車程」或「省 ¥3000 住宿」有感。

👨‍👩‍👧 全家行程主規劃者

家中那位「負責安排所有事」的人。需要把方案印給長輩看、需要算總預算給配偶看、需要在現場把備案攤開來決定。

📋 小型旅遊團 / 朋友揪團

透過分享連結讓其他成員直接編輯，不需要註冊帳號。團員可以即時看到誰改了哪一格。

🇰🇷 重視當地圖資的使用者

例如去韓國的人會切換到 Kakao Map（韓國的本地地圖龍頭），因為 Google 在韓國境內的大眾運輸班次與路線細節較弱。

03 · 設計原則

產品哲學

五條主張，貫穿每一個按鈕、每一個欄位、每一次決策。

編輯器把時間軸 + 地圖 + 浮動景點卡合在同一頁，並讓使用者可調整三者寬度。永遠不需要在 tab 之間切換才看到完整景象。

拖完景點、改完時間、刪一個方案，立刻寫進資料庫。介面用 optimistic update 立即反映，後端非同步落地。除非要刪掉整趟旅程，否則沒有「確定／取消」對話框打斷流程。

沒有 hero banner、沒有插畫、沒有「立即註冊」按鈕。首頁就是儀表板，列出你的旅程。模板與品牌語言參考 Cal.com — 白底、黑色按鈕、Inter 字型、極簡 hairline 邊框、少數 pastel 強調色用在類別徽章。

內部用「貨幣品牌型別」強制每一筆金額帶幣別，杜絕「把日圓當台幣加總」的 bug。但 UI 上不會問使用者「你確定要繼續嗎？」— 衝突的時段顯示紅框警告，不阻擋操作。

每場旅行都可以一鍵匯出成 PDF 手冊（A4 / A5 / Letter，直橫向）。封面、目錄、全趟地圖、Day spread、行李清單、費用總表、票卷附頁，全部按使用者勾選的章節生成。

05 · 資料模型

核心概念：旅程 → 方案 → 日 → 排程項目

在系統內部，所有東西都圍繞著這條主軸組織。理解這四層階層，就理解了 80% 的產品邏輯。

Trip 旅程
→
Plan 方案 (多個)
→
Day 日
→
ScheduleItem 排程項目
→
Place 地點

主要實體

主表

Trip 旅程

一趟旅行的最外層容器

· 標題、起訖日
· 主幣別（TWD / JPY / KRW …）
· 封面圖示與顏色
· 擁有者 + 成員

Plan 方案

同一趟可有多個獨立版本

· 例：輕鬆版 / 緊湊版
· 步調設定（輕鬆 / 標準 / 緊湊）
· 可從另一個 plan 複製

Day 日

方案內每一天的容器

· 日期
· 該日筆記

ScheduleItem 排程

當天的「一件事」

· 種類：景點/餐廳/住宿/航班...
· 起訖時間
· 順序索引
· 各種類自訂欄位（如航班號）

Place 地點

全 DB 共享的地點快取

· 來自 Google / Kakao / 手動
· 座標、評價、類型、營業時間
· 韓國地點額外有韓文名稱

Transport 交通

連接相鄰兩個排程的線段

· 模式：駕車/大眾/步行/飛行
· 距離、時間、路線編碼
· 預估費用

Ticket 票券

掛在某個排程上的票

· 類別、金額、訂位代碼
· 1:1 自動同步到費用

Expense 費用

會計層的單筆花費

· 類別、金額、幣別
· 自動 或 手動
· 自動的有來源標記

AISuggestion AI 建議

LLM 為該 plan 生成的內容

· 種類：注意事項 / 行李清單
· 快取避免重複呼叫

TripShare 分享連結

讓他人加入的邀請鏈接

· token 雜湊儲存
· 角色：編輯 / 檢視
· 可設過期、可撤銷

Settings 設定

每位使用者一筆

· API 金鑰（加密儲存）
· 預設幣別、油價
· LLM provider 清單

ApiUsageLog 用量

記每次外部 API 呼叫

· 服務、token、估算成本
· 供儀表板統計
重要：方案是獨立的不是分支 在系統內，一個 Plan 是完整且獨立的一份排程。複製方案時會深拷貝所有 Day、ScheduleItem、Transport、Expense — 修改 A 方案不會影響 B 方案。Place 則跨方案共享快取（同一個景點不會抓兩次）。

關鍵元素

  • 頂部狀態列：4 個關鍵指標（進行中／完成／本年花費／分享中），讓使用者一眼掌握全局。
  • 「繼續上次編輯」：最近一次開啟的旅程置頂，避免使用者要找一陣子。
  • 旅程卡片：圖示式封面（不用真實照片避免授權問題），標題 + 起訖日 + 方案數 + 分享狀態。
  • ＋ 新增旅程：彈窗式表單，填標題、日期、主幣別、目的地圖示即可完成。新增後自動建立一個「主要方案」與對應天數。
  • 「匯入旅程」（隱藏式）：可貼一段文字（從別處複製的行程），LLM 解析成完整結構。
頂部狀態列指標的意義 「進行中／已完成／本年累計／分享中」這四個數字不只是裝飾。它們是產品給使用者的近期紀錄回顧：今年走過多少地方、花了多少錢、有多少場行程仍在規劃中。讓使用者每次進入都能感覺到旅遊紀錄的累積。

07 · 主要頁面

行程編輯器 — 應用的心臟

使用者花最多時間的頁面。三大區塊：頂部日期帶、左側時程清單、右側地圖。所有編輯都在這完成，不跳頁。

縱向時間軸卡片

每個排程項目以卡片顯示：類別圖示、時間、地點、評分、價格。卡片間有「交通段」橫條：「步行 12 分 · 950 m」或「駕車 28 分 · 18 km · NT$ 180 油費」。

  • 類別自動換 icon：景點 = 地標、餐廳 = 刀叉、住宿 = 床、機場 = 飛機。
  • 時間衝突（前一段結束時間 > 下一段開始時間）顯示紅色邊框，但不阻擋操作。
  • 拖曳卡片可在當日內或跨日重排。
  • Shift+click 方案標籤可多選方案，自動切換到「列表內方案對比」模式（並列顯示，地圖收折）。

浮動景點卡（Floating Place Card）

點任何排程項目（或地圖上的標記），跳出可任意拖曳的詳情卡。它是 fixed + Portal 渲染 — 不受任何父容器邊界限制，可拖到整個視窗任何位置。

  • 顯示：地點名稱（中英對照 + 韓文若有）、評分、類別 badge、地址、營業時間、電話、照片。
  • 編輯：可修改顯示名稱（不會覆蓋 Google 原始名稱）、預設停留時間、自訂備註、上傳照片。
  • 「重新綁定到其他地點」：跳出地點搜尋彈窗，可選 Google 或 Kakao 來源。
  • 關閉方式：拖出視窗外、ESC、點右上角 ✕、或點地圖空白處。

交通編輯對話框

點兩個排程項目之間的「→ 交通段」橫條打開。包含三大區：

  1. 模式切換：駕車 / 大眾運輸 / 步行 / 自行車 / 飛行 / 自訂。每個模式有不同的詳情面板（例：駕車面板秀油費試算、收費路段、休息站；飛行面板秀航班號、航空公司、機場代碼）。
  2. 地圖整合面板：以分頁切換顯示 Google Maps / Kakao Maps 路線。可直接貼上一段 Google Maps 或 Kakao Map 的詳細路線文字，LLM 自動解析成站名 / 時間 / 票價（支援中／英／日／韓，韓文站名輸出為中韓對照 "新沙站 (신사역)"）。
  3. 路線方案選擇器（Google Routes API 提供時）：列出 2–3 個替代路線，比較時間 / 距離 / 預估費用 / 交通量級。使用者點選後鎖定，重排景點不重算這段。

範圍選擇器

頂部 pill 切換器決定對比的「時間範圍」：

  • 整趟旅程 — 所有日子加總，看總時間 / 總花費。
  • 單天 — 選某一天，並排看三個方案那天怎麼安排。
  • 區間 — 選 Day 2–Day 5，比較中段差異。

下方除了並列卡片，還有指標總覽表：每一行是一個維度（總時間 / 景點數 / 餐費 / 住宿費 / 交通費 / 步行公里 / 駕車公里…），最便宜或最少的用淺綠強調。

09 · 主要頁面

費用總覽

不只是清單，而是「自動結算 + 手動補登」的會計層。

兩種費用來源

AUTO

自動費用

系統從排程項目、票券、交通段自動產生。包含：景點門票、餐廳預算、住宿總價、駕車油費、大眾運輸票價、機票。修改來源時自動重算。

MANUAL

手動費用

使用者親手新增，例如：紀念品、現場才知道的入場費、突發狀況的花費。重算引擎不會動到這些列。

介面組成

  • 總計儀表：四個大數字 — 總花費（主幣）、平均每日花費、最大單筆、最大類別佔比。
  • 分類橫條圖：食 / 宿 / 交通 / 票券 / 購物 / 雜項，看比例分布。
  • 明細清單：可按日 / 按類別 / 按方案分組。每列顯示金額 + 原始幣 + 換算成主幣 + 自動/手動標籤。
  • 多幣別顯示：所有金額顯示為 "NT$ 78,400 · ¥ 373,184"（主幣 + 灰色當地幣）。
  • 方案切換器：頁面頂部有 pill 切換器，可在不同方案的費用之間跳轉，直接看每個方案花多少錢。
  • 幣別拆分表（多幣別行程時）：列出每個原幣別的小計，方便核對信用卡帳單。

10 · 主要頁面

手冊匯出 (PDF)

把整趟旅程印成一本可帶上路的手冊。網路斷了、手機沒電了，紙本永遠有。

控制面板

  • 紙張大小：A4 / A5 / Letter（mm-based 真實比例）
  • 方向：直向 / 橫向（真正重新排版，不是縮放）
  • 字級：小 / 中 / 大（倍率調整）
  • 配色：彩色 / 單色（為了省墨水）
  • 章節 toggle：可開關每一個章節
  • 預估頁數：即時顯示 "約 14 頁"

章節清單（可勾選）

📕 封面

標題、起訖日、主規劃者、印製日期

📑 目錄

章節導引 + 頁碼

🗾 全趟地圖

所有景點打點 + 編號 + 連線

📋 行前注意事項

AI 生成 — 簽證、貨幣、緊急聯絡、習俗、季節

🎒 行李清單

AI 生成 — 衣物、文件、3C、藥品、洗漱、應急

📅 Day Spread

每天一個對開頁：時程表 + 當日地圖 + 餐廳

💴 費用總表

分類匯總 + 每日小計 + 總計

🎟 票卷附頁

所有票券的訂位代碼、QR、附件

📖 封底

備用聯絡、保險資訊、版本資訊

11 · 主要頁面

AI 建議

把 LLM 用在「規劃前才有意義」的位置。不是聊天機器人，是針對性的內容生成。

📋

行前注意事項

根據旅行天數、目的地氣候、活動類型（自駕／自助／親子），列出衣物層、3C 配件、文件、應急品。可勾選追蹤打包進度。

停留時間優化

對任一景點，AI 重新評估「合理停留時間」（考慮類型、評分、規模）。Heuristic 預設值 + AI 微調，使用者可覆寫。

🚗

交通建議 (Suggest Transport)

在交通對話框內的「AI 自動填」按鈕。AI 根據起終點、日期、地區、旅程步調，建議最合理的交通方式 + 時長 + 預估費用 + 大眾運輸線路名稱。

LLM Provider 抽象 產品不綁定特定 AI 商。使用者在設定中可加入多個 provider（OpenAI / Anthropic / Google / 本地 Ollama …），填 API Key 與模型，並設定預設。每次呼叫會記錄 token 與估算成本。

分享對話框

  • 角色：編輯者（可改）或 檢視者（唯讀）
  • 標籤：「給家人」、「團員 A」— 方便日後追蹤誰透過哪條連結加入
  • 過期時間：可選 7 天 / 30 天 / 永不
  • 撤銷：一鍵撤銷某連結，並可一併剔除透過該連結加入的成員
  • 成員列表：顯示誰加入了、上次活動時間、角色

並發處理

同一個 plan 被多人同時編輯時，Day 物件帶版本號。其中一人提交時若版本不符，會跳「衝突」訊息要求使用者重新整理，不會默默蓋掉別人改的內容。

即時 Presence Indicator

編輯器頂部會顯示「目前還有誰在看這個旅程」的小頭像列。每 15 秒輪詢一次，看到的是 lastSeenAt 在最近 30 秒內的成員。不是 WebSocket、不是即時游標，是輕量的「我也在現場」訊號。

13 · 主要頁面

設定中心

所有外部服務的金鑰、預設值、備份功能集中在這頁。

設定區塊

淺色 / 深色 / 跟隨系統。即時切換不需重整。

支援 OpenAI / Anthropic / Google / 自訂 baseURL。每個 provider 可定義多個 model。可手動測試連線（呼叫一次小請求驗證）。設預設 provider + model。

API Key 用 AES-256-GCM 加密後寫進 DB，永遠不會明文外洩到前端。

三選一：Google Maps（最完整）／ Mapbox（最美）／ OSM（免費）。Settings 內填對應 API Key + Map ID。應用會根據此設定切換編輯器內的地圖元件。

兩把金鑰：

  • JavaScript Key — 渲染 Kakao 地圖（需在 developers.kakao.com 註冊 Web 平台 domain）
  • REST API Key — 伺服器端搜尋韓國 POI（韓國地點覆蓋率優於 Google）

填入後，韓國行程的搜尋預設改用 Kakao，並可在路線詳情切換到 Kakao 顯示。

填入航班 API 金鑰。新增 FLIGHT 排程時，輸入航班號可自動帶入航空公司、出發機場、抵達機場、班表。
設定主幣別（TWD）、預設當地幣（JPY）。匯率表為手動或定時抓取的 JSON 快照。每筆 Expense 落地時記錄當下匯率，避免日後匯率變動影響歷史紀錄。
每公升油價（如 NT$ 35）+ 每公升公里數（如 15 km/L）。駕車段距離自動換算油費。
國際 / 國內航班的 check-in 緩衝（預設 120 / 60 分）、入境緩衝（60 / 30 分）。新增航班時自動插入「報到」與「入境」項目，使用者可手動覆寫。
一鍵匯出整個 DB 成 JSON 檔，包含所有 Trip / Plan / Day / 設定（API Key 不含）。可匯入回相同或另一個帳號。也是換主機或備份意外的保險。
掃描資料庫找出沒主人的 Trip / Plan / Day（migration 或人為錯誤導致），可選擇認領或刪除。是進階管理員工具。

列出每月所有外部 API 呼叫的統計：呼叫次數、token 用量、估算的 USD 成本。分服務看（Google Places autocomplete / Directions / Static Maps、LLM Chat、AviationStack、AeroDataBox…）。

可設「每月軟性預算上限」(USD)，達 80% 時警告、超過時顯示紅色警示，但不阻擋繼續使用。

顯示目前登入狀態（admin / guest）。Admin 模式可輸入密碼解鎖固定 30 天的「admin」user ID（避免換瀏覽器就遺失資料）。Guest 模式則綁定當前瀏覽器 cookie。

14 · 核心系統

多方案系統

這是這個產品最重要的差異化功能。一趟旅行不只有一個版本，使用者可以平行設計、比較、決定。

使用情境

  • 步調對比：輕鬆版（每天 2-3 個景點）vs 緊湊版（每天 5-6 個）
  • 預算對比：高鐵版 vs 自駕版 vs 巴士版，看到底差多少錢
  • 主題對比：美食版 vs 自然版 vs 文化版
  • 陪伴對象：帶小孩版 vs 兩人輕旅行版

關鍵互動

  • 複製方案：以現有方案為基底深拷貝一份，再開始改差異點。
  • 部分複製：從 A 方案的 Day 3 把幾個項目複製進 B 方案的 Day 5。
  • List 內方案對比：Shift+click 多個方案 pill，自動進入並列模式，地圖收折，可以拖某個項目從 A 方案複製到 B 方案。
  • 方案上限：3 個。經產品決策定為限制，避免使用者陷入過度比較癱瘓。
  • 預設方案：Trip 有 defaultPlanId 欄位，記錄使用者最常開啟的方案，下次進 trip 直接打開。

15 · 核心系統

地圖整合 — 多供應商策略

不綁死任一家。使用者選擇符合需求與預算的供應商。

供應商強項弱項適用情境
Google Maps全球景點與評價最完整需付費 + 韓國大眾運輸不準大部分國家的預設
Mapbox樣式美觀、可自訂POI 資料比 Google 少追求視覺效果的使用者
OSM (OpenStreetMap)免費 + 開源樣式陽春、無評價不想付費、自用為主
Kakao Maps韓國境內最完整僅韓國 + 需註冊 domain韓國行程切換用

地圖面板的共通功能

  • 編號標記：每個景點以編號徽章顯示（1, 2, 3...），對應左側時程順序。
  • 路線連線：交通段渲染為實線（已確認模式）或虛線（free 模式）。
  • 自動 fit bounds：當天景點 + 邊界自動填滿視窗。
  • 地點搜尋疊層：搜尋輸入框，輸入時即時 autocomplete。可在 Google / Kakao 之間切換來源。
  • 點地圖加入：點地圖任一點，跳出「加入這個地點到」彈窗，選日期與類別即可加入排程。

16 · 核心系統

交通與路線

不只是「兩點之間的距離」，而是完整考慮成本、時間、模式、實際路線的子系統。

6 種交通模式

🚗 駕車 DRIVING

距離換算油費（每升油價 × km / 油耗），可選收費路段、休息站。LLM 可分析路線並列出「平面 / 收費 / 高速」分段。

🚇 大眾運輸 TRANSIT

支援貼上 Google Maps 或 Kakao 的詳細路線文字，LLM 解析成站名、線色、班次、票價。Korean 輸出採中韓對照（"新沙站 (신사역)"）。

🚶 步行 WALKING

適用短距離。自動秀「12 分 · 950 m」橫條。

🚲 自行車 BICYCLING

同步行但用自行車速度。

✈ 飛行 FLIGHT

連同 ScheduleItem 的 FLIGHT 一起記錄。航班號、座位、行李、報到/入境緩衝。

🔧 自訂 CUSTOM

不套用任何規則，使用者自填時間、距離、費用、備註。

特殊狀態：Free 模式

Phase 14p 後預設新建的交通段為「free」狀態 — 沒指定模式、沒計算時間。在週視圖中以虛線顯示、不佔時間軸。使用者明確選擇模式後才會變成實線並計時。

LLM 貼文字解析器

交通對話框內的「貼入區」是這個產品最聰明的工具之一。使用者把 Google Maps / Kakao Map / 任何旅遊網站複製來的一段路線文字（任何語言），貼進去後選：

  • Rule 優先：先用 regex 規則抓總時長 / 票價 / 線名。失敗才打 LLM。
  • AI 優先：直接用 LLM 結構化輸出，速度慢但細節最完整。

輸出結果自動填入：總時長、票價、各段轉乘步驟、線色、站名。使用者按「套用」即寫進交通段。

Google Routes API 整合

當 Google Maps Key 有設定時，系統可呼叫 Routes API 拿到 2–3 個替代路線，並列比較：時間 / 距離 / 預估費用 / 駕車塞車量級 / 大眾運輸換乘次數。使用者點選後鎖定為 manuallyEdited，重排不重算。

17 · 核心系統

費用與貨幣

這是產品內部「最容易出 bug 但最不能出錯」的子系統。一個美元當台幣加進去，結果差 30 倍。

貨幣品牌型別 (Money branded type)

系統內部把所有金額包成 Money<C> 物件，永遠帶幣別。任何加總、換算操作都必須先轉到同一幣別。這在工程層消除了「把日圓當台幣加總」這類經典 bug。對產品經理而言只需知道：每一筆金額永遠知道自己的幣別，UI 永遠以主幣 + 當地幣雙顯示。

顯示規則

  • 主幣在上、當地幣在下灰色小字：NT$ 78,400 · ¥ 373,184
  • 不使用 .k 縮寫（避免 NT$ 78.4k 不夠精確）
  • 僅在 ≥ 100 萬時才使用 .M
  • 每筆 Expense 寫入時記錄當下的 fxRateToBase，未來匯率變動不會回頭改歷史

自動費用來源

來源說明

TRANSPORT_FARE大眾運輸／高鐵／巴士的票價
TRANSPORT_FUEL駕車段的油費（自動換算）
ATTRACTION_TICKET景點門票（從 metadataJson 抓）

19 · 視覺系統

深淺色主題

所有顏色用 CSS 變數定義，深色主題透過 [data-theme="dark"] 覆寫變數。即時切換不需重整。試試右上角的「切換深色」按鈕。

主題的設計準則

  • 不是反白：深色主題的 surface 階梯保留與淺色相同的層次感（canvas → soft → card → strong）。
  • 品牌色固定：4 個 pastel badge 在兩個主題都是同樣的 hex，但深色主題下背景帶 alpha 透明度。
  • 強調色不變：brand-accent 藍（#3B82F6）在兩個主題完全一致。
  • 陰影調整：深色主題用更黑的陰影（深色背景上白色陰影看不見）。
